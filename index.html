Constants

Constants are block-scoped, much like variables defined using the let statement. The value of a constant cannot change through reassignment, and it can’t be redeclared.

const MY_CONST = 6;

console.log(MY_CONST);

//12

MY_CONST = 8;

//Silent error, as we cannot reassign a new value to a constant

In reality, const is not really a constant and it’s not immutable.

const obj = {
 one: 1,
 two: 2
};

// Object mutation is ok
// {one: 1, two: 2, three: 3}
obj.three = 3;

Reassignment doesn't work

// Uncaught TypeError: Assignment to constant variable.
obj = {
 one: "bazinga",
 two: "bazinga"
};

Arrow function

ECMA Script 5

var add = function(a, b) {
 return a + b;
};

ECMA Script 6+

const add = (a, b) => a + b;

Template Literals

ECMA Script 5

var name = 'Your name is ' + first + ' ' + last + '.'
var url = 'http://localhost:3000/api/messages/' + id

ECMA Script 6+

let name = `Your name is ${first} ${last}.`
let url = `http://localhost:3000/api/messages/${id}`

Classes in ES6

class Person {
 constructor(name) {
 this.name = name;
 }
 describe() {
 return `Person called ${this.name};
 }
}

Derived classes

class Employee extends Person {
 constructor(name, title) {
 super(name);
 this.title = title;
 }
 describe() {
 return `${ super.describe() }: ${ this.title }`;
 }
}
